API Documentation

Comprehensive REST API for course comparison using advanced MCDM algorithms

Quick Start

Our API provides comprehensive endpoints for course search, comparison, and ranking using TOPSIS, AHP, and personalized MCDM algorithms.

Base URL:
https://your-domain.com/api
Authentication:

Currently, the API is open and doesn't require authentication. Future versions will include JWT-based authentication.

Response Format:

All responses are in JSON format with appropriate HTTP status codes.

API Endpoints

GET
/api/courses/search

Search courses by keyword with optional MCDM algorithm selection

Parameters:
  • keyword (required): Search term for courses
  • algorithm (optional): MCDM algorithm (TOPSIS, AHP, PERSONALIZED)
Example Request:
GET /api/courses/search?keyword=python&algorithm=TOPSIS
POST
/api/courses/advanced-search

Advanced search with multiple filters and criteria

Request Body:
{ "keyword": "machine learning", "filters": { "minPrice": 0, "maxPrice": 100, "minRating": 4.0, "platform": "Coursera", "difficulty": "Intermediate" }, "algorithm": "TOPSIS" }
POST
/api/courses/rank

Rank courses using specified MCDM algorithm

Request Body:
[ { "id": 1, "title": "Python Course", "rating": 4.5, "price": 29.99 } ]
GET
/api/courses/platform/{platform}

Get courses from a specific platform

Example:
GET /api/courses/platform/Coursera
GET
/api/courses/page

Get courses with pagination and sorting

Parameters:
  • page: Page number (default: 0)
  • size: Page size (default: 20)
  • sortBy: Sort field (default: rating)
  • sortDir: Sort direction (asc/desc, default: desc)

MCDM Algorithms

TOPSIS

Technique for Order of Preference by Similarity to Ideal Solution. Provides comprehensive multi-criteria decision making.

  • Normalizes criteria values
  • Calculates ideal solutions
  • Ranks by similarity
AHP

Analytic Hierarchy Process. Uses pairwise comparisons to determine criteria weights and course rankings.

  • Pairwise comparisons
  • Consistency checking
  • Weighted scoring
Personalized

Custom algorithm that considers user preferences, interests, and learning history for personalized recommendations.

  • User preferences
  • Interest matching
  • Learning history
